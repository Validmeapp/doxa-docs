# Document Validation Request

A Document Validation Request allows users to accept and submit their documents for validation. The organization sends a request asking the user to share documents, and the user can choose to accept and send them.

## Request Body

```json
{
  "flowId": "flow-id-12345",
  "phoneNumber": "+59166666666",
  "description": "ACME Corp needs to validate your identity",
  "acceptanceMode": "BUTTON",
  "category": "DOCUMENT_VALIDATION",
  "callbackUrls": ["https://your-app.com/webhook"],
  "metadata": {
    "customerId": "123"
  }
}
```

## Request Parameters

### Required Parameters

- **flowId** (required): The unique identifier of your verification flow
- **phoneNumber** (required): User's phone number in international format (e.g., +59166666666)
- **description** (required): Clear and simple description explaining why the organization needs this verification
- **acceptanceMode** (required): How the user will accept the request (OTP, CHOICE, BUTTON)
- **category** (required): Must be "DOCUMENT_VALIDATION"

### Optional Parameters

- **metadata** (optional): Custom metadata (e.g., customerId from your system)
- **callbackUrls** (optional): URLs where ValidMe will send webhook events


## Response Fields

- **_id**: Unique request identifier
- **status**: Current request status (PENDING, CONFIRMED, SENDING, SUCCESS, REJECTED, etc.)
- **verifications**: Array of document verification statuses
- **description**: The description sent with the request
- **createdAt**: When the request was created
- **statusTransitions**: History of status changes
