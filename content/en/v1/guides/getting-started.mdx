# Getting Started

This comprehensive guide will help you integrate with our digital identity platform API in minutes. You'll learn how to authenticate, make your first API calls, and implement common workflows.

## Prerequisites

Before you begin, make sure you have:

- A developer account (sign up at [developer.example.com](https://developer.example.com))
- Basic knowledge of REST APIs and HTTP requests
- Your preferred programming language and HTTP client

## Step 1: Get Your API Keys

1. **Log in** to your developer dashboard
2. **Navigate** to the "API Keys" section
3. **Click** "Generate New Key"
4. **Copy** your API key and store it securely

```bash
# Your API key will look like this:
sk_live_1234567890abcdef1234567890abcdef
```

## Step 2: Make Your First API Call

Let's start by testing your API key with a simple request to get your account information:

```bash
curl -X GET "https://api.example.com/v1/account" \
  -H "Authorization: Bearer sk_live_1234567890abcdef1234567890abcdef" \
  -H "Content-Type: application/json"
```

```javascript
// Using fetch in JavaScript

    'Content-Type': 'application/json'
  }
```

## Step 3: Create Your First User

Now let's create a user account in your application:

```javascript
const userData = {
  email: 'john.doe@example.com',
  first_name: 'John',
  last_name: 'Doe',
  password: 'SecurePassword123!',
  send_welcome_email: true
};

const response = await fetch('https://api.example.com/v1/users', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(userData)
});

const user = await response.json();
console.log('User created:', user);
```

## Step 4: Set Up Webhooks (Optional)

Webhooks allow you to receive real-time notifications when events occur. Here's how to set up a basic webhook:

```javascript
const webhookData = {
  url: 'https://your-app.com/webhooks/identity',
  events: ['user.created', 'user.verified', 'identity.verified'],
  description: 'Main webhook for user events'
};

const response = await fetch('https://api.example.com/v1/webhooks', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(webhookData)
});

const webhook = await response.json();
console.log('Webhook created:', webhook);
```

### Webhook Endpoint Example

```javascript
const express = require('express');
const crypto = require('crypto');
const app = express();

app.use(express.raw({ type: 'application/json' }));

app.post('/webhooks/identity', (req, res) => {
  const signature = req.headers['x-webhook-signature'];
  const payload = req.body;
  
  // Verify webhook signature
  const expectedSignature = crypto
    .createHmac('sha256', process.env.WEBHOOK_SECRET)
    .update(payload, 'utf8')
    .digest('hex');
  
  if (`sha256=${expectedSignature}` !== signature) {
    return res.status(401).send('Invalid signature');
  }
  
  const event = JSON.parse(payload);
  console.log('Received event:', event.type);
  
  // Handle the event
  switch (event.type) {
    case 'user.created':
      console.log('New user:', event.data.user.email);
      break;
    case 'user.verified':
      console.log('User verified:', event.data.user.email);
      break;
    default:
      console.log('Unhandled event type:', event.type);
  }
  
  res.status(200).send('OK');
});

app.listen(3000, () => {
  console.log('Webhook server running on port 3000');
});
```

## Step 5: Handle Errors Gracefully

Always implement proper error handling in your integration:

```javascript
async function createUser(userData) {
  try {
    const response = await fetch('https://api.example.com/v1/users', {
      method: 'POST',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(userData)
    });
    
    if (!response.ok) {
      const error = await response.json();
      throw new Error(`API Error: ${error.error.message}`);
    }
    
    return await response.json();
  } catch (error) {
    console.error('Failed to create user:', error.message);
    throw error;
  }
}
```

```python
import requests
from requests.exceptions import RequestException

def create_user(user_data):
    try:
        response = requests.post(
            'https://api.example.com/v1/users',
            headers={
                'Authorization': 'Bearer YOUR_API_KEY',
                'Content-Type': 'application/json'
            },
            json=user_data,
            timeout=30
        )
        
        response.raise_for_status()
        return response.json()
        
    except requests.exceptions.HTTPError as e:
        error_data = e.response.json()
        print(f"API Error: {error_data['error']['message']}")
        raise
    except RequestException as e:
        print(f"Request failed: {str(e)}")
        raise
```

## Common Error Codes

| Code | Description | Solution |
|------|-------------|----------|
| `401` | Unauthorized | Check your API key |
| `403` | Forbidden | Verify your permissions |
| `404` | Not Found | Check the endpoint URL |
| `422` | Validation Error | Review request data |
| `429` | Rate Limited | Implement retry logic |
| `500` | Server Error | Contact support |

## Rate Limits

Our API implements rate limiting to ensure fair usage:

- Free tier**: 1,000 requests per hour
- Pro tier**: 10,000 requests per hour  
- Enterprise**: Custom limits

### Handling Rate Limits

```javascript
async function makeAPIRequest(url, options) {
  const response = await fetch(url, options);
  
  if (response.status === 429) {
    const retryAfter = response.headers.get('Retry-After');
    console.log(`Rate limited. Retry after ${retryAfter} seconds`);
    
    // Wait and retry
    await new Promise(resolve => setTimeout(resolve, retryAfter * 1000));
    return makeAPIRequest(url, options);
  }
  
  return response;
}
```
