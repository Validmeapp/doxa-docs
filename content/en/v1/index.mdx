# ValidMe: Decentralized Identity Validation

Welcome to **ValidMe**, the modern platform for validating identities securely, quickly, and in a decentralized manner.

## What is ValidMe?

ValidMe is a SaaS platform that enables companies to verify the identity of their users through:

- **Document Validation**: Capture and analysis of identifications
- **Biometric Verification**: Face validation through selfies
- **Decentralized Access**: Secure verifications without storing sensitive data
- **Regulatory Compliance**: Complies with KYC/AML regulations

### Main Features

| Feature | Description |
|---|---|
| **E2E Encrypted** | End-to-end encryption with RSA |
| **Real-time Webhooks** | Instant event notifications |
| **Flexible Flows** | Create custom validation workflows |
| **API First** | Complete programmatic integration |
| **Multi-country Support** | Document validation from multiple countries |
| **Intuitive Dashboard** | Full control over validations |

## How Does It Work?

### Basic Flow

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Your app creates a validation request                    │
├─────────────────────────────────────────────────────────────┤
│ 2. User receives link/SMS with QR code                      │
├─────────────────────────────────────────────────────────────┤
│ 3. User uploads document + takes selfie in ValidMe app      │
├─────────────────────────────────────────────────────────────┤
│ 4. ValidMe processes and validates documents                │
├─────────────────────────────────────────────────────────────┤
│ 5. Your app receives webhook with result (encrypted)        │
├─────────────────────────────────────────────────────────────┤
│ 6. Your app decrypts data and grants access                 │
└─────────────────────────────────────────────────────────────┘
```

## Key Concepts

### Request (Validation Request)
An individual validation initiated by your company. Each request contains:
- Unique ID
- Required verifications (documents, selfie, etc)
- Current status
- Custom metadata

### Flow (Validation Workflow)
A reusable template that defines:
- Which documents are required
- Valid countries
- Validation rules

**Example**: "Bolivia KYC Flow" requires ID card + selfie

### Verification
A specific type of validation within a request:
- NATIONAL_ID (National identity card)
- SELFIE (Face validation)
- SELFIE_VIDEO (Video with instructions)
- Others (based on regional availability)

### Webhook
HTTP POST notification sent to your server when:
- Request status changes
- A document is uploaded
- A verification is completed
