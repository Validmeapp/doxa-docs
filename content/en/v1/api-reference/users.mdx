# Users API

The Users API allows you to manage user accounts and profiles in your application. This API provides full CRUD operations with advanced filtering, pagination, and search capabilities.

## Base URL

```
https://api.example.com/v1/users
```

## Authentication

All endpoints require authentication via API key or OAuth 2.0 token.

```bash
Authorization: Bearer YOUR_API_KEY
```

## Endpoints

### GET /users

Retrieve a paginated list of users with optional filtering.

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `page` | integer | No | Page number (default: 1) |
| `limit` | integer | No | Items per page (default: 20, max: 100) |
| `search` | string | No | Search users by name or email |
| `status` | string | No | Filter by status: `active`, `inactive`, `pending` |
| `created_after` | string | No | ISO 8601 date filter |

#### Request Example

```bash
curl -X GET "https://api.example.com/v1/users?page=1&limit=10&status=active" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

#### Response Example

```json

    {
      "id": "usr_1234567890",
      "email": "john.doe@example.com",
      "first_name": "John",
      "last_name": "Doe",
      "status": "active",

      "updated_at": "2024-01-15T10:30:00Z",
      "profile": {
        "avatar_url": "https://example.com/avatars/john.jpg",
        "phone": "+1234567890",
        "timezone": "America/New_York"
      }
    }
  ],
  "pagination": {
    "page": 1,

    "total_pages": 15
  }
}
```

### POST /users

Create a new user account with email verification.

#### Request Body

```json
{
  "email": "jane.smith@example.com",
  "first_name": "Jane",
  "last_name": "Smith",
  "password": "SecurePassword123!",
  "profile": {
    "phone": "+1987654321",
    "timezone": "America/Los_Angeles"
  },
  "send_welcome_email": true
}
```

#### Request Example

```javascript
const response = await fetch('https://api.example.com/v1/users', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'jane.smith@example.com',
    first_name: 'Jane',
    last_name: 'Smith',
    password: 'SecurePassword123!',
    profile: {
      phone: '+1987654321',
      timezone: 'America/Los_Angeles'
    },
    send_welcome_email: true
  })
});

const user = await response.json();
```

#### Response Example

```json

  }

```

### GET /users/{id}

Retrieve a specific user by their unique identifier.

#### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | User ID (format: `usr_*`) |

#### Request Example

```python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.example.com/v1/users/usr_1234567890',
    headers=headers
)

user = response.json()
```

#### Response Example

### PUT /users/{id}

Update a user's information. Only provided fields will be updated.

#### Request Body

```json
{
  "first_name": "Jonathan",
  "profile": {
    "timezone": "Europe/London",
    "preferences": {
      "notifications": false
    }
  }
}
```

#### Request Example

```bash
curl -X PUT "https://api.example.com/v1/users/usr_1234567890" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "Jonathan",
    "profile": {
      "timezone": "Europe/London"
    }
  }'
```

### DELETE /users/{id}

Permanently delete a user account. This action cannot be undone.

#### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `force` | boolean | No | Skip soft delete and permanently remove |

#### Request Example

```javascript
const response = await fetch('https://api.example.com/v1/users/usr_1234567890', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

if (response.ok) {
  console.log('User deleted successfully');
}
```

#### Response

```json
{
  "message": "User deleted successfully",
  "deleted_at": "2024-01-20T15:30:00Z"
}
```

## Error Responses

All endpoints return consistent error responses:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "The provided email address is already in use",
    "details": {
      "field": "email",
      "value": "john.doe@example.com"
    }
  }
}
```

### Common Error Codes

- `VALIDATION_ERROR` - Invalid input data
- `NOT_FOUND` - User not found
- `UNAUTHORIZED` - Invalid or missing authentication
- `FORBIDDEN` - Insufficient permissions
- `RATE_LIMITED` - Too many requests
