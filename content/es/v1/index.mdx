# ValidMe: Validación de Identidad Descentralizada

Bienvenido a **<span className="brand-name">ValidMe</span>**, la plataforma moderna para validar identidades de manera segura, rápida y descentralizada.

## ¿Qué es ValidMe?

ValidMe es una plataforma SaaS que permite a empresas verificar la identidad de sus usuarios mediante:

- **Validación de documentos**: Captura y análisis de identificaciones
- **Verificación biométrica**: Validación de rostro mediante selfies
- **Acceso descentralizado**: Verificaciones seguras sin almacenar datos sensibles
- **Cumplimiento normativo**: Cumple con regulaciones KYC/AML

### Características Principales

| Característica | Descripción |
|---|---|
| **E2E Encrypted** | Encriptación de extremo a extremo con RSA |
| **Real-time Webhooks** | Notificaciones instantáneas de eventos |
| **Flexible Flows** | Crea flujos personalizados de validación |
| **API First** | Integración programática completa |
| **Soporte Multipaís** | Validación de documentos de múltiples países |
| **Dashboard Intuitivo** | Control total de validaciones |

## Casos de Uso

### 1. **Fintech & Banca Digital**
Valida identidades en segundos para apertura de cuentas.

```
Usuario → ValidMe → Aprobación inmediata
```

### 2. **Marketplaces**
Verifica vendedores antes de permitirles listar productos.

```
Registro → Validación → Acceso a vendedor
```

### 3. **Plataformas de Viajes**
KYC obligatorio para reservas internacionales.

```
Reserva → Validación → Confirmación
```

### 4. **Casinos & Juegos**
Cumplimiento AML/KYC requerido.

```
Registro → Validación completa → Acceso
```

## ¿Cómo Funciona?

### Flujo Básico

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Tu App crea una solicitud de validación                 │
├─────────────────────────────────────────────────────────────┤
│ 2. Usuario recibe link/SMS con código QR                   │
├─────────────────────────────────────────────────────────────┤
│ 3. Usuario carga documento + toma selfie en app ValidMe    │
├─────────────────────────────────────────────────────────────┤
│ 4. ValidMe procesa y valida documentos                     │
├─────────────────────────────────────────────────────────────┤
│ 5. Tu app recibe webhook con resultado (encriptado)        │
├─────────────────────────────────────────────────────────────┤
│ 6. Tu app desencripta datos y otorga acceso                │
└─────────────────────────────────────────────────────────────┘
```

✅ **Confiable**: 99.9% uptime  
✅ **Escalable**: Procesa millones de validaciones

## Conceptos Clave

### Request (Solicitud)
Una validación individual iniciada por tu empresa. Cada request contiene:
- ID único
- Verificaciones requeridas (documentos, selfie, etc)
- Estado actual
- Metadatos personalizados

### Flow (Flujo)
Plantilla reutilizable que define:
- Qué documentos se requieren
- En qué países son válidos
- Reglas de validación

**Ejemplo**: "Flujo KYC Bolivia" requiere cédula + selfie

### Verification (Verificación)
Un tipo específico de validación dentro de un request:
- NATIONAL_ID (Cédula de identidad)
- SELFIE (Validación de rostro)
- SELFIE_VIDEO (Video con instrucciones)
- Otros (según disponibilidad regional)

### Webhook
Notificación HTTP POST enviada a tu servidor cuando:
- Cambia el estado de una solicitud
- Se carga un documento
- Se completa una verificación
